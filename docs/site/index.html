Recovery modes

Three ways to recover.
All time-locked.

Every recovery mode stages the key rotation behind a configurable timelock. The real owner can always cancel during the window.

✉

Email + World ID

Owner sends a recovery email. A Noir ZK circuit proves the DKIM signature is valid without revealing the email address. World ID proves the initiator is human. Both proofs verified on-chain.

ZK + WORLD ID
🔑

Backup wallet

A pre-registered backup address can initiate recovery directly. No proofs needed — just a transaction from the backup wallet. Simplest mode for solo operators.

SINGLE TX
👥

Social / Guardian

M-of-N guardians vote to approve a new owner. Once the threshold is met, the timelock starts automatically. Nonce increments invalidate stale votes on cancel.

M-OF-N
How it works

Register, recover,
execute

Spectre does not deploy agents — it protects existing ones. Register a recovery config, then recover when you need to.

01

Register

Owner registers a recovery config for their agent wallet. On-chain: SHA256 of recovery email, timelock duration, optional guardians and backup wallet.

02

Initiate recovery

Send a recovery email with the new key + nonce in the body. Generate a ZK proof of the DKIM signature. Verify with World ID. Submit both proofs on-chain.

03

Execute

After the timelock expires, anyone can finalize. The agent record gets the new owner. Nonce increments. The old key loses control.

email + world id recovery flow
Owner sends email: body = {newKey}:{nonce} | v Relayer parses .eml, extracts DKIM signature | v Noir circuit proves DKIM valid + email hash matches + body = recovery params | v Owner scans World ID QR (proves human, binds signal to recovery) | v SpectreRegistry.initiateRecovery() verifies both proofs on-chain | v Timelock starts (~24h on mainnet) | +-- Owner cancels --> recovery cleared, nonce + 1 | +-- Timelock elapses --> executeRecovery() rotates owner key
Architecture

Four components,
one protocol

A Noir ZK circuit, Solidity contracts, a TypeScript relayer, and a World ID frontend — each independently verifiable.

circuits/

Noir ZK circuit for DKIM email verification. Proves RSA-2048 signature validity, FROM address hash, and body content — all without revealing the email. Compiled to UltraHonk proofs via Barretenberg.

contracts/

Solidity smart contracts on Base. SpectreRegistry manages agent records and enforces recovery rules. Verifier.sol (auto-generated) verifies UltraHonk proofs on-chain. Built with Foundry.

relayer/

TypeScript off-chain service. Parses .eml files, fetches DKIM keys from DNS, generates ZK witnesses, builds proofs via nargo + bb, and submits transactions. Full CLI for all recovery operations.

world-id-ui/

React frontend for World ID proof generation. User fills in recovery parameters, scans with World App, and gets a Semaphore proof. Signal binds the proof to the specific recovery (owner + newOwner + nonce).

ZK circuit

What the proof verifies

The Noir circuit generates a zero-knowledge proof that the recovery email is authentic — without revealing the email address or contents.

RSA-2048 DKIM signature
Proves the email was signed by the sender's mail server. The RSA public key is a public input verified against DNS.
FROM address hash
SHA256 of the sender's email address matches the on-chain commitment. The actual address is never revealed on-chain.
Body content binding
Email body must contain exactly {recovery_key}:{nonce}, binding the proof to a specific recovery attempt.
Body hash integrity
The DKIM bh= header matches the SHA256 of the canonicalized body, proving the body hasn't been tampered with.
CLI reference

Developer commands

All recovery operations are available through the relayer CLI. Run from the relayer/ directory.

Command Description
npm run register Register agent with email hash + timelock
npm run initiate Initiate Email + World ID recovery
npm run cancel Cancel pending recovery (owner only)
npm run execute Execute recovery after timelock elapses
npm run check View agent record and recovery status
npm run set-backup Set backup wallet address
npm run initiate-backup Initiate backup wallet recovery
npm run set-guardians Configure guardian addresses + threshold
npm run approve-guardian Guardian votes for a recovery
npm run server Start the HTTP prover API
Prover API

HTTP endpoints

The relayer server exposes a REST API for proof generation and World ID context signing. Runs on port 3001.

POST /prove
Upload a .eml file with recovery params. Returns the ZK proof and public inputs as hex.
POST /verify
Verify a proof against public inputs and verification key.
POST /worldid-context
Get a signed RP context for the World ID widget. Required before opening the IDKit modal.
GET /health
Health check. Returns { ok: true }.
