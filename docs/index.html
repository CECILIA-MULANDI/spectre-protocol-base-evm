Recovery modes

Four ways to recover. All time-locked.

Every recovery mode stages the key rotation behind a configurable timelock. The real owner can always cancel during the window. Recovery is never instant.

01

ZK Email + World ID

ZK proof of DKIM signature. World ID proves the initiator is human. Both proofs verified on-chain.

ZK + WORLD ID
02

Backup wallet

Pre-registered backup address initiates recovery. No proofs needed. Simplest mode for solo operators.

SINGLE TX
03

Social / Guardian

M-of-N guardians vote to approve. Timelock starts when threshold is met. Nonce increments invalidate stale votes on cancel.

M-OF-N
04

Humanity proof (pluggable)

World ID today, any ZK humanity proof tomorrow. Identity layer is intentionally modular.

PLUGGABLE
How it works

Register once. Recover when you need to.

Spectre does not deploy agents. It protects existing ones. Register a recovery config, then recover when you need to.

01

Register

Owner registers a recovery config for their agent wallet. On-chain: SHA256 of recovery email, timelock duration, optional guardians and backup wallet.

02

Initiate recovery

Send a recovery email with the new key + nonce in the body. Generate a ZK proof of the DKIM signature. Verify with World ID. Submit both proofs on-chain.

03

Execute

After the timelock expires, anyone can finalize. The agent record gets the new owner. Nonce increments. The old key loses control.

email + world id recovery flow
Owner sends email: body = {newKey}:{nonce} | v Relayer parses .eml, extracts DKIM signature | v Noir circuit proves DKIM valid + email hash matches + body = recovery params | v Owner scans World ID QR (proves human, binds signal to recovery) | v SpectreRegistry.initiateRecovery() verifies both proofs on-chain | v Timelock starts (~24h on mainnet) | +-- Owner cancels --> recovery cleared, nonce + 1 | +-- Timelock elapses --> executeRecovery() rotates owner key

Architecture

Four components. One protocol.

A Noir ZK circuit, Solidity contracts, a TypeScript relayer, and a World ID frontend. Each independently verifiable.

circuits/

Noir DKIM circuit

Proves RSA-2048 signature validity, FROM address hash, and body content, without revealing the email. Compiled to UltraHonk proofs via Barretenberg.

contracts/

Solidity contracts

SpectreRegistry manages agent records and enforces recovery rules. HonkVerifier verifies UltraHonk proofs on-chain. Built with Foundry.

relayer/

TypeScript relayer + CLI

Parses .eml files, fetches DKIM keys from DNS, generates ZK witnesses, builds proofs via nargo + bb, submits transactions.

world-id-ui/

World ID frontend

React UI for World ID proof generation. Signal binds the proof to the specific recovery (owner + newOwner + nonce).

What the proof verifies

RSA
RSA-2048 DKIM signature

Proves the email was signed by the sender's mail server. Public key verified against DNS.

@
FROM address hash

SHA256 of the sender's email matches the on-chain commitment. The actual address is never revealed.

{ }
Body content binding

Email body must contain exactly {recovery_key}:{nonce}, binding the proof to one specific recovery attempt.

#
Body hash integrity

The DKIM bh= header matches SHA256 of the canonicalized body. The body cannot be tampered with.

CLI reference

Developer commands

All recovery operations are available through the relayer CLI. Run from the relayer/ directory.

Command Description
npm run register Register agent with email hash + timelock
npm run initiate Initiate Email + World ID recovery
npm run cancel Cancel pending recovery (owner only)
npm run execute Execute recovery after timelock elapses
npm run check View agent record and recovery status
npm run set-backup Set backup wallet address
npm run initiate-backup Initiate backup wallet recovery
npm run set-guardians Configure guardian addresses + threshold
npm run approve-guardian Guardian votes for a recovery
npm run server Start the HTTP prover API

HTTP endpoints

POST /prove Upload a .eml file with recovery params. Returns the ZK proof and public inputs as hex.
POST /verify Verify a proof against public inputs and verification key.
POST /worldid-context Get a signed RP context for the World ID widget. Required before opening the IDKit modal.
GET /health Health check. Returns { ok: true }.
